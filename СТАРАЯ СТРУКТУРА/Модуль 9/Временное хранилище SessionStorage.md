## Описание

sessionStorage – это встроенное в браузер хранилище, которое **сохраняет данные только на время сессии**. Данные удаляются, как только вкладка или браузер закрывается.

> [!info] **Простыми словами:**
>   Это временное хранилище в браузере, которое исчезает при закрытии вкладки. Полезно для временных данных, которые не нужно хранить долго.

---
## Как работает sessionStorage?

- **Данные доступны только пока открыта вкладка**.
- **Объем хранилища – около 5MB**.
- **Не передается серверу**, доступно только в браузере.
- **Отдельное хранилище для каждой вкладки** – данные не переносятся между вкладками.

---
## Когда использовать sessionStorage?

✔ **Временные данные** (например, статус формы или текущий шаг регистрации).  
✔ **Защита данных от перезагрузки страницы** (например, фильтры поиска, настройки отображения).  
✔ **Хранение данных, которые не должны сохраняться после закрытия вкладки**.
❌ **Не использовать для важных данных!** (`sessionStorage` очищается при закрытии вкладки, и данные могут потеряться).

---
## Основные методы sessionStorage

### 1. Просмотр хранилища

```js {7}
sessionStorage.setItem("user-id", "123");
sessionStorage.setItem("tickets", JSON.stringify({ from: "Lviv", to: "Kyiv", quantity: 2 }));

console.log(sessionStorage);
// Storage {user-id: '123', tickets: '{"from":"Lviv","to":"Kyiv","quantity":2}', length: 2}
```

---
### 2. Сохранение данных

```js
sessionStorage.setItem("username", "Alice");
```

> [!done]
> Теперь в `sessionStorage` сохранится ключ `username` со значением `Alice`.

---
### 3. Получение данных

Методом `getItem(key)` можна читати записи, використовуючи збережений ключ.

```js
const user = sessionStorage.getItem("username");
console.log(user); // Alice
```

---
### 4. Удаление конкретного элемента

```js
sessionStorage.removeItem("username");
```

---
### 5. Очистка всего sessionStorage

```js
sessionStorage.clear();
```

> [!warning]
> После `clear()` **все данные в sessionStorage удалятся!**

---
## Работа с объектами в sessionStorage

> [!info]
> `sessionStorage` хранит **только строки**, поэтому **объекты и массивы нужно преобразовывать** в JSON.

### 1. Сохранение объекта

```js
const user = { name: "Alice", age: 25 };
sessionStorage.setItem("user", JSON.stringify(user));
```

### 2. Чтение объекта

```js
const userData = JSON.parse(sessionStorage.getItem("user"));
console.log(userData.name); // Alice
```

> [!done]
> Теперь `userData` – это объект, с которым можно работать.

---
## Проверка наличия данных в sessionStorage

```js
if (sessionStorage.getItem("user")) {
    console.log("Данные есть!");
} else {
    console.log("Данных нет!");
}
```

> [!warning]
> Если ключ `user` существует, выведется `Данные есть!`.

---
## ИТОГ:

✔ `sessionStorage` – это временное браузерное хранилище, данные которого **удаляются при закрытии вкладки**.  
✔ Доступ к данным возможен **только с клиентской стороны**.  
✔ Данные хранятся в виде **строк**, но можно использовать `JSON.stringify()` и `JSON.parse()`.  
✔ Подходит для **временных данных (например, текущего состояния UI, форм, фильтров)**.

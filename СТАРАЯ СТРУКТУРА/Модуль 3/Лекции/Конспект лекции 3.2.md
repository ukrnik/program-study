## Лекция
### [[Function]]

Самое главное отличие **function declaration** от **function expression** это то что к **function declaration** мы можем обращаться из любой точки кода а **function expression** можно обратится только снизу функции 

```js
// ❌ Помилка! Не працює виклик до оголошення
multiply(1, 2, 3);

const multiply = function (x, y, z) {
  console.log(x * y * z);
};

// ✅ Працює виклик після оголошення
multiply(4, 5, 6);
```

### Псевдомасив [[arguments]]

Это зминна которая позволяет принять все переменные которые мы передали в тело функции. arguments автоматически создается и создаёт псевдомасив из переданых переменных

```js
function foo(){
	console.log(arguments); // [1, 2, 3]
}

foo(1, 2, 3);
```

В псевдомасиве мы можем проверить только длину псевдомасива а также использовать цикл for и доставать значения через квадратные скобки []

```js
function foo(){
	console.log(arguments); // [1, 2, 3]
	console.log(arguments.length); // 3
	
	for (let i = 0; i < arguments.length; i++){
		const element = arguments[i];
		console.log(element); // 1, // 2, // 3
	}
}

foo(1, 2, 3);
```

### Конструктор [[Array.from()]]

Помогает нам сделать перевод с псевдомасива в масив

```js
function foo(){
	console.log(arguments); // [1, 2, 3] ❌ нет нужных методов
	
	const realArray = Array.from(arguments);
	
	console.log(realArray); // [1, 2, 3] ✅ это нормальный масив с                                                      привычними методами
}

foo(1, 2, 3);
```

### Stack вызовов

**Стек** — структура даних, яка працює за принципом LIFO (Last-In-First-Out), тобто останнім прийшов — першим вийшов. Останнє, що додається на стек, буде видалено з нього першим, значить, можна додати або видалити елементи тільки з верхівки стека. Уяви стек як масив, у якого можна додати або видалити тільки елемент у кінці колекції.

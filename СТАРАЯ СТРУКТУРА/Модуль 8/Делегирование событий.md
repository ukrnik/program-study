## Описание
### Что такое делегирование событий?

<font color="#409CFF">`fas:InfoCircle`</font> Делегирование событий — это техника обработки событий, при которой мы назначаем обработчик не на сами элементы, а на их родителя. Это позволяет обрабатывать события для множества элементов динамически, даже если они добавляются позже.

![[Pasted image 20250204023819.png]]
### Почему это полезно?

- <font color="#409CFF">`fas:InfoCircle`</font> Меньше обработчиков – экономим память, не вешая обработчики на каждый элемент отдельно.
- <font color="#409CFF">`fas:InfoCircle`</font> Работает с динамическими элементами – подходит для элементов, которые добавляются после загрузки страницы.
- <font color="#409CFF">`fas:InfoCircle`</font> Простота в управлении – легче менять логику обработки событий.
### Как работает делегирование событий?

Мы назначаем один обработчик событий на родительский элемент. Когда событие всплывает (например, click), мы проверяем, на каком элементе оно произошло, и выполняем нужное действие.

---
## <font color="#f71735">`ris:Alert`</font> Когда не стоит использовать делегирование событий?

<font color="#f71735">`ris:Alert`</font> Когда элементы имеют слишком сложную структуру и вложенность.
<font color="#f71735">`ris:Alert`</font> Когда обработчик должен реагировать только на конкретный элемент, а не на всех потомков.
<font color="#f71735">`ris:Alert`</font> Когда событий слишком много (например, mousemove), и это может привести к тормозам.

## Пример №1: Делегирование событий на списке

```html
<ul id="menu">
    <li>Элемент 1</li>
    <li>Элемент 2</li>
    <li>Элемент 3</li>
</ul>
```

```js
    document.getElementById("menu").addEventListener("click", function(event) {
        // Проверяем, кликнули ли именно на <li>
        if (event.target.tagName === "LI") {
            alert("Вы выбрали: " + event.target.textContent);
        }
    });
```

✅ **Как это работает?**

1. Назначаем обработчик click на `<ul id="menu">`.
2. Проверяем, что клик был на `<li>`, а не на самом `<ul>`.
3. Показываем alert с текстом выбранного элемента.

## Пример №2: Делегирование для динамических элементов

```html
<button id="addItem">Добавить элемент</button>
<ul id="menu"></ul>
```

```js
    document.getElementById("menu").addEventListener("click", function(event) {
        if (event.target.tagName === "LI") {
            alert("Вы кликнули по: " + event.target.textContent);
        }
    });

    document.getElementById("addItem").addEventListener("click", function() {
        let newItem = document.createElement("li");
        newItem.textContent = "Новый элемент";
        document.getElementById("menu").appendChild(newItem);
    });
```

✅ **Как это работает?**

1. Кнопка Добавить элемент создаёт новый `<li>`.
2. Обработчик на `<ul>` всё равно реагирует на клики по новым `<li>`, так как слушает события от всех потомков.

## ИТОГ:

✔ Делегирование событий позволяет устанавливать обработчик на родительский элемент, а не на каждый потомок отдельно.

✔ Это удобно для работы с динамическими элементами.

✔ Нужно быть внимательным, чтобы правильно фильтровать события.

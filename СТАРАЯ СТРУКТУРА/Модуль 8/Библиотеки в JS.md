## Описание

**Библиотека** – это набор готовых функций, классов и модулей, которые помогают разработчикам ускорить процесс написания кода, избегая повторного изобретения уже существующих решений.

<font color="#409CFF">`fas:InfoCircle`</font> Простыми словами:

 - **Библиотека** – это как сборник инструментов для разработчика, который упрощает работу.

> [!info] **Основные характеристики библиотек:**
> **1. Многократное использование**: Библиотеки создаются для повторного использования. Разработчики могут интегрировать библиотеки в разные проекты и использовать их функционал без необходимости писать код с нуля.
> **2. Переопределенный функционал**: Библиотеки содержат готовый код, который можно использовать для решения общих задач. Например, библиотеки могут содержать функции для работы с датами, обработки строк, выполнения анимаций и многое другое.
> **3. Ускорение разработки**: Использование библиотек позволяет сократить время разработки, поскольку разработчику не нужно создавать и отлаживать функции или компоненты с нуля. Они могут использовать готовые и проверенные решения.
> **4. Общество и поддержка**: Хорошие библиотеки часто поддерживаются активным сообществом разработчиков, которые вносят улучшение, исправление ошибок и обеспечивают обратную совместимость.

## Синтаксис

Библиотека содержит заранее написанный код, который можно импортировать и использовать в своем проекте.

Пример использования **библиотеки Math** в JavaScript:

```js
let num = Math.sqrt(25); // Используем метод sqrt() из библиотеки Math
console.log(num); // 5
```

✅ Мы не писали сами функцию для квадратного корня – просто взяли её из Math.

## Подключение библиотеки

Чтобы иметь возможность использовать библиотеку в проекте, ее необходимо подключить, то есть добавить файлы библиотеки к проекту.

Для примера подключим к проекту библиотеку [Lodash](https://lodash.com/) через CDN. Подключение JavaScript библиотеки через CDN состоит из нескольких шагов.

**Шаг 1.** Для начала зайди на сайт CDN сервиса https://www.jsdelivr.com/ и найди необходимую библиотеку по ее имени.

![[Pasted image 20250204000926.png|600]]

**Шаг 2.** В списке результатов выберите необходимую библиотеку, кликнув на название. После перехода по ссылке ты окажешься на странице с информацией о библиотеке.

![[Pasted image 20250204001025.png|600]]

**Шаг 3.** Скопируй HTML-код тега script, после чего открой свой HTML-файл и добавь ссылку на скрипт библиотеки в конце HTML-документа, как показано в примере.

```html
<!DOCTYPE html>
<html lang="en">
  <head>
	 <!-- head tags -->
  </head>
  <body>
    <!-- HTML-markup -->

    <!-- Lodash library script file -->
		<script async src="https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js"></script>
    <!-- Your script file -->
    <script defer src="path/to/script.js"></script>
  </body>
</html>

```

> [!warning]
> - Подключение скрипта библиотеки должно быть до подключения твоего основного файла скриптов.
> - Не забудь добавить тег script библиотеки async. Это нужно для того, чтобы файл библиотеки загружался как можно скорее.

При подключении библиотек через CDN к объекту window добавляется свойство, в котором сохраняется то, что предоставляет библиотека. Имя этого свойства уникально для библиотеки и описано в документации.

Для [[Lodash]] – это символ нижнего подчеркивания`_`.

```js
console.log(_); // Весь объект библиотеки Lodash
```

<font color="#fcb97d">`ris:Question`</font> Для проверки используй методы:

- [sum](https://lodash.com/docs/4.17.15#sum) — он вычисляет сумму значений в массиве
- [shuffle](https://lodash.com/docs/4.17.15#shuffle) — создает массив перетасованных значений

```js
console.log(_.sum([4, 2, 8, 6])); // 20
console.log(_.sum([5, 10])); // 15

console.log(_.shuffle([1, 2, 3, 4])); // [4, 1, 3, 2]
console.log(_.shuffle([1, 2, 3, 4])); // [3, 2, 1, 4]

```

> [!attention]
> **Обрати внимание!** Способ подключения и использования библиотек может зависеть от конкретной библиотеки.

## Разница между библиотекой и фреймворком

| Характеристика       | Библиотека                              | Фреймворк                                                 |
| -------------------- | --------------------------------------- | --------------------------------------------------------- |
| Кто управляет кодом? | Разработчик вызывает функции библиотеки | Фреймворк управляет кодом и вызывает функции разработчика |
| Гибкость             | Можно использовать только нужные части  | Навязывает структуру проекта                              |
| Пример               | jQuery, Lodash, Axios                   | React, Angular, Vue                                       |

<font color="#409CFF">`fas:InfoCircle`</font> Простыми словами:

- Библиотека – это набор инструментов, который ты используешь по своему усмотрению.
- Фреймворк – это среда, которая диктует, как ты должен писать код.

## Пример работы с Axios (запрос на сервер):

```js

axios.get('https://jsonplaceholder.typicode.com/todos/1')
    .then(response => console.log(response.data))
    .catch(error => console.error(error));
```

DOM — это структура, которая описывает HTML-документ как дерево объектов.

Document Object Model или ([[DOM]]) - это независимый функционал что соединяет роботу станицы html и JS. С помощь DOM мы можем искать, удалять, создавать елементы html и менять значения под нужды пользователя. Структура DOM это каждый елемент html (абзац, ссылка, заголовок, span и тд.) и все они могут быть измененнми с помощь JavaScript. 
## HTML-документ и DOM
В DOM-модели каждый тег html создаёт отдельный element node а текст внутри тега это text node

HTML-документ — это иэрархичное дерево, в котором каждый элемент (кроме коренного) вложенный в один родительский элемент.
![[DOMTree.jpg]]

```html
<!DOCTYPE html>
<link lang="en">
<link>
	<meta charset="UTF-8">
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
	<title>Document</title>
	<link></link>
</head>
<body>
	<div>
		<img src="" alt="">
		<h1></h1>
		<p></p>
		<div></div>
	</div>
	<script>
	</script>
</body>
</html>
```

Document - это последний коренной элемент.
Html вложенный внутрь Document и поэтому (Document это отцовский элемент для html)

## Как складывается DOM-дерево

Чтобы отобразить HTML-документ, браузер сначала изменяет его в формат, который он понимает — DOM. Интерпретатор браузера имеет специальный фрагмент кода — `HTML-парсер`, который используется для изменения HTML в DOM.

Браузер строит DOM постепенно, как только поступают первие фрагменты кода, он начиет парсить HTML, добавляючи узлы в деревоподобную структуру.

![[HtmlParser.gif]]
После того как DOM-дерево построено, в нём можна найти элементы с помощью JavaScript. Поскольку каждый элемент имеет интерфейс из безконечных свойств и методов, с них можна использовать необходимые действия.

## Навигация в DOM

Элементы DOM-дерева имеют иерархическое отношение друг к другу. Для описания отношений используются термины:
- предок (ancestor)
- отпрыск (descendant)
- отец (parent)
- ребенок (child)
- сосед (sibling).

- Самый высокий элемент называется корневым (root node).
- Каждый элемент кроме корневого имеет только один родительский.
- У элемента может быть сколько угодно детей.
- Соседи – это элементы с общим родительским элементом.
- Дочерние элементы (дети) — элементы непосредственно вложенные в текущий элемент (элементы первого уровня вложенности).
- Потомки — все элементы, которые находятся в текущем элементе вместе с их детьми, детьми их детей и т.д. То есть все вроде дерева.

`<ul>` – это родительский элемент. Три элемента `<li>` – дети для `<ul>` и соседи между собой. Так же элементы `<span>, <a>, <p>` соседствуют. Все они – дети для `<li>` и потомки для `<ul>`.

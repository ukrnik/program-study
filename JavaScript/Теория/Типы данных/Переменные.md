## Переменная - это контейнер который создаётся для хранения данных в ячейке памяти и для дальнейшего использования

Переменная состоит из:
	- Идентификатора (уникального имени);
	- Области памяти, где сохраняется ее значение.

![[Pasted image 20241130001156.png]]


```js
const age = 20;
const username = "Mango";
```

---
### 1. var

• **Область видимости (scope)**: Ограничивается функцией, в которой объявлена переменная, либо глобальным пространством, если она объявлена вне функции.

• Переменные, объявленные через var, **не ограничены блочной областью видимости** (например, в циклах или if).

• **Подъем (hoisting)**: Переменные, объявленные через var, поднимаются в верх области видимости и инициализируются как undefined.

• **Перезапись**: Можно переопределить переменную без ошибки.

• **Особенность**: Допускает повторное объявление переменной.

Пример:

```js
console.log(x); // undefined
var x = 10;
if (true) {
  var y = 20; // доступна за пределами блока
}
console.log(y); // 20
```

---
### 2. let

• **Область видимости (scope)**: Ограничивается блоком кода {}, в котором объявлена переменная.

• Это делает let более безопасным для использования в современных проектах.

• **Подъем (hoisting)**: Переменные, объявленные через let, также поднимаются, но остаются **неинициализированными** до их явного объявления. Это приводит к **ошибке при доступе до объявления** (Temporal Dead Zone - временная мёртвая зона).

• **Перезапись**: Значение можно изменить, но нельзя повторно объявить ту же переменную в одной области видимости.

Пример:

```js
console.log(x); // ReferenceError: Cannot access 'x' before initialization
let x = 10;
if (true) {
  let y = 20; // доступна только внутри блока
  console.log(y); // 20
}
console.log(y); // ReferenceError: y is not defined
```

---

### <font color="#1ac8ed">3. const</font>

• **Область видимости (scope)**: Как и let, ограничивается блоком кода {}.

• **Подъем (hoisting)**: Поведение идентично let. Переменная остаётся в состоянии “мёртвой зоны” до инициализации.

• **Изменение значения**: Нельзя изменить значение, присвоенное переменной. Однако, если это объект или массив, их содержимое **может изменяться**.

• **Особенность**: Обязательно требуется **инициализация при объявлении**.

Пример:

```js
const x = 10;
x = 20; // TypeError: Assignment to constant variable

const arr = [1, 2, 3];
arr.push(4); // допустимо, потому что изменяется содержимое массива
console.log(arr); // [1, 2, 3, 4]
```

## Переопределение значения

Чтобы перезаписать значение в переменной нужно после названия переменной через = написать новое значение.

```js
let username = "Mango";
username = "Poly";
```

Для объявления переменной, которой дальше можно задать новое значение, используется ключевое слово `let`

```js
let username = "Mango";
console.log(username); // "Mango"

username = "Poly";
console.log(username); // "Poly"
```

`fas:ExclamationTriangle`<font color="#d99694">Если переменная объявлена как `const` то будет ошибка так переопределить ее значение невозможно.</font>

```js
const username = "Mango";
console.log(username); // "Mango"

// ❌ Неправильно, буде помилка
username = "Poly"; // TypeError: Assignment to constant variable.
```

Создание переменной без ключевого слова let или const в режиме strict mode приведет к ошибке.

Переменным, объявленным через let, необязательно сразу задавать значение. В случае объявления переменной без значения переменная инициализируется со специальным значением undefined.

```js
let username;
console.log(username); // undefined

username = "Mango";
console.log(username); // "Mango"
```

## Названия для переменных

1. Имена могут состоять из букв (a-z, A-Z), цифр (0-9), символов __ и знака доллара ($).
   
2. Первым символом в имени переменной должна быть буква латиницы, символ подчеркивания или знак доллара. Другие символы могут быть любыми из допустимых.

3. Имена переменных чувствительны к регистру, то есть переменные user, usEr и User считаются разными переменными.

В JavaScript мы используем camelCase, первое слово пишется строчными буквами, а каждое следующее начинается с прописной буквы, например: user, getUserData, isActive, activeGuestCount. Не забывай isActive и IsActive – это разные переменные.
